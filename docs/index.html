What is CloudCommerce?

CloudCommerce is a comprehensive multi-tenant e-commerce platform engineered for scale. From a single deployment you can provision, customize, and operate hundreds of independent storefronts — each with its own domain, branding, product catalog, payment configuration, and customer base.

Built with modern tooling — Next.js 16, React 19, TypeScript 6, Prisma 7, and PostgreSQL — the platform delivers enterprise-grade features out of the box: real-time order tracking, Redis-powered caching, BullMQ background jobs, WebSocket push, pluggable payment/storage/shipping providers, and a full REST API.

Every tenant is isolated at the data layer. Every operation is audited. Every component is built for extensibility. Whether you're launching a SaaS, running a digital agency, or building the next great marketplace, CloudCommerce gives you complete ownership of your infrastructure and data.

Architecture Patterns

Designed for scale, isolation, and maintainability from day one.

Multi-Tenant

Subdomain-based routing with tenantId on all data models. Each tenant is fully isolated at the database layer.

Repository Pattern

BaseRepository<T> provides consistent CRUD with automatic audit logging for every mutation.

Service Layer

Business logic lives in src/lib/services/ — 32 service modules called by route handlers, not duplicated.

Provider Pattern

Pluggable architecture for Payments, Storage, Tax, Shipping, and Email — swap implementations via factory functions.

Feature Gating

Plan-based controls (Starter / Growth / Enterprise) unlock capabilities per tenant without code changes.

Event-Driven

Redis Pub/Sub powers real-time order events, notification dispatch, and cross-service communication.

Queue-Driven

BullMQ handles async work — email delivery, inventory sync, webhook dispatch — with retries and dead-letter queues.

Caching Layer

Namespaced Redis cache with automatic key invalidation. Supports Upstash, Redis Cloud, and localhost.

Soft Deletes

Products, Categories, and Variants use deletedAt nullable columns — non-destructive with full audit trail.

Pagination

Both offset-based and cursor-based pagination available across all list endpoints.

Features

Every capability your SaaS e-commerce platform needs, built and ready.

Store Management

Create and configure unlimited stores with full branding control and custom domains.

  • Custom domains with automatic SSL
  • Branding customization — fonts, colors, logos
  • Subdomain-based routing per tenant
  • Plan-based feature gating
  • Staff management with granular permissions

Product Management

Full product lifecycle management with variants, images, and SEO metadata.

  • Product variants with SKU tracking
  • Configurable options (size, color, etc.)
  • Product images with gallery support
  • Category tree hierarchy
  • Bulk import / export
  • SEO metadata per product
  • Review and rating aggregation

Inventory Management

Global and per-warehouse tracking with reservations, alerts, and audit trails.

  • Global + per-warehouse inventory counts
  • Stock reservations for active carts
  • Low-stock thresholds and automated alerts
  • Full audit trail (who changed what and when)
  • Inter-warehouse stock transfers
  • Auto-sync via BullMQ inventory worker

Shopping Cart

Guest and authenticated carts with seamless merge on login.

  • Guest carts stored via localStorage
  • Authenticated carts persisted server-side
  • Automatic cart merge on login
  • Real-time pricing calculations
  • Cart persistence across sessions

Order Processing

Complete order lifecycle with real-time WebSocket events and discount support.

  • Lifecycle: pending → payment → processing → shipped → delivered → cancelled / refunded
  • Guest checkout and account orders
  • Order items with variant snapshots
  • Discount and promotion application
  • Real-time events via WebSocket
  • Tax calculation at order level

Payment Processing

Multi-provider payment support with a pluggable architecture.

  • Stripe — credit cards, SEPA, iDEAL
  • Khalti — Nepal payment gateway
  • eSewa — Nepal payment gateway
  • Pluggable provider pattern for custom integrations
  • Payment verification and refund processing
  • Webhook handling with retry queue
  • Subscription billing via Stripe

Customer Management

Full customer profiles with address books, order history, and segmentation.

  • Customer accounts with profiles
  • Address book — multiple addresses per customer
  • Complete order history per customer
  • Customer groups and segments
  • Notification preferences per channel
  • Loyalty account integration

Reviews & Ratings

1–5 star product ratings with moderation workflow and verified purchase badges.

  • 1–5 star ratings with title, body, and images
  • Verified purchase badge
  • Moderation workflow — pending / approved / hidden
  • Helpfulness votes on reviews
  • Merchant replies to reviews
  • Average rating aggregation per product

Wishlists

Per-customer wishlists with sharing and move-to-cart functionality.

  • Authenticated wishlists (1 per customer)
  • Guest wishlists via session cookie
  • Shareable via unique token
  • Move-to-cart in one click
  • Public shared wishlist views

Discounts & Promotions

Flexible coupon codes and automatic promotions with rule-based conditions.

  • Coupon codes — fixed amount, percentage, free shipping
  • Automatic promotions with conditions
  • Rules: min amount, product, category, customer
  • Stacking with priority levels
  • Usage tracking per coupon / promotion
  • Date range limits and first-order-only offers

CMS — Content Management

Drag-and-drop section blocks for editable storefront pages with SEO metadata.

  • Editable storefront pages via admin
  • Section blocks: hero, text, image, product grid, category grid, banner, CTA
  • Page-level SEO metadata
  • Home page flag support
  • Separate admin editor and storefront renderer per block type

Analytics & Reporting

Dashboard widgets and exportable reports with plan-tiered depth.

  • Basic analytics on Starter, advanced on Growth+
  • Charts via recharts — line, bar, area
  • Dashboard widgets: metric, chart, area, bar cards
  • Time-based filtering
  • Revenue, orders, customer metric exports

Staff Management

Role-based access control with granular permissions and audit logging.

  • Roles: owner (full CRUD), staff (CRU), customer (none)
  • Granular permissions: create, read, update, delete, archive, manage
  • Staff invitations via email
  • Activity audit logging per user

API & Webhooks

Full REST API v1 covering 36 resource modules with outbound webhook delivery.

  • REST API v1 — products, orders, customers, and more
  • API key authentication (Enterprise feature)
  • x-tenant-id header for multi-tenant scoping
  • Outbound webhooks with retry queue
  • Webhook event types for all major actions
  • Dead-letter queues for failed deliveries

Shipping

Multi-method shipping with zone-based rates and carrier extensibility.

  • Flat rate, weight-based, price-based shipping
  • Free shipping (conditional) defaults
  • Zone-based rate calculation — countries, states, zip codes
  • Carrier registration system (extensible for UPS / FedEx / DHL)
  • Per-warehouse shipping configuration

Tax Calculation

Built-in and third-party tax providers with compound rate support.

  • Built-in zone-based tax (country / state / region)
  • TaxJar third-party provider
  • Avalara third-party provider
  • Compound tax rate support
  • Tax exemption handling
  • Provider factory pattern

Loyalty Program

Points-based loyalty with tiers, multipliers, and a reward rules engine.

  • Points on purchases, reviews, referrals
  • Tier system — Bronze → Silver → Gold
  • Tier-based point multipliers
  • Points expiration support
  • Reward rules engine (event-driven)
  • Transaction history with balance tracking

Warehouse Management

Multi-warehouse support with per-location inventory and inter-warehouse transfers.

  • Multi-warehouse with physical locations
  • Per-warehouse inventory counts
  • Stock reservations per warehouse
  • Inter-warehouse stock transfers
  • Transfer status: pending, completed, cancelled
  • Warehouse geolocation data

Real-time System

WebSocket server with auto-reconnect, heartbeat, and history replay.

  • WebSocket server on port 3001
  • Event types: order.created, .payment_received, .shipped, .cancelled
  • Heartbeat keepalive every 30s
  • Auto-reconnect with exponential backoff
  • History replay on reconnect (last 50 events)
  • Redis Pub/Sub for server-to-server events

Notifications

Multi-channel notifications via WebSocket and email with per-event preferences.

  • In-app notifications via WebSocket push
  • Email notifications via BullMQ queue
  • Per-channel per-event preferences
  • Inventory alerts — low stock, out of stock
  • Order notifications — created, paid, shipped, cancelled
  • Unread count tracking

Background Processing

BullMQ queues for mail, inventory, notifications, and webhooks with dead-letter management.

  • Mail queue — 5 retries, exponential backoff
  • Inventory queue — 3 retries
  • Notifications queue — 3 retries
  • Webhooks queue — 5 retries
  • Dead-letter queues with 30-day retention
  • Monitoring dashboard with queue metrics
  • Retry failed jobs from dead letter

File Uploads & Storage

Auto-selecting storage provider with image optimization and tenant isolation.

  • Auto-select: R2 → S3 → Local
  • Image validation — JPEG / PNG / WebP, max 10MB
  • Auto thumbnails: small 150×150, medium 400×400, large 800×800
  • WebP output via sharp
  • Per-tenant upload isolation

Search

Cached product search with category filtering and auto-invalidation.

  • Redis-cached product search
  • Category-based search filtering
  • Search result caching with TTL
  • Cache invalidation on product / category updates

Authentication & Authorization

Custom session-based auth with PBKDF2 hashing and role-based permissions.

  • Custom session-based auth — no third-party
  • PBKDF2 SHA-512 password hashing (100k iterations)
  • httpOnly session cookies (cc_session_token)
  • 7-day session expiry with DB storage
  • Roles: owner (full CRUD), staff (CRU), customer (none)
  • CSRF token protection via x-csrf-token header
  • API key authentication for Enterprise plan

Security

Defense-in-depth with CSP, rate limiting, CSRF protection, and secure headers.

  • Content Security Policy headers
  • Rate limiting — 100 req/min/IP, Redis-backed
  • CSRF protection
  • HSTS preload, X-Frame-Options DENY
  • Security headers — X-Content-Type-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy
  • Session cookie security — httpOnly, secure in prod, sameSite=lax

Deployment & Infrastructure

Dockerized with CI/CD pipelines and one-command cloud deployment scripts.

  • docker-compose with PostgreSQL 16

  • Redis auto-connect: Upstash → Redis Cloud → localhost
  • Prisma migrations and seed scripts
  • Health check API endpoint

│   ├── ui/             # shadcn/ui primitives (48)
│   ├── cc/views/       # Page-level view components
│   ├── dashboard/      # Dashboard widgets
│   └── storefront/     # Storefront components

├── lib/
│   ├── services/       # Business logic (32 modules)
│   ├── payments/       # Stripe, Khalti, eSewa
│   ├── security/       # CSRF, rate limiting
│   ├── upload/         # Storage providers
│   ├── webhooks/       # Webhook processing
│   └── queue/          # BullMQ queue definitions
├── workers/          # BullMQ worker processors
├── data/             # Mock data fallback
├── proxy.ts           # CSP, auth, rate-limiting
└── ws-server.ts       # WebSocket server
